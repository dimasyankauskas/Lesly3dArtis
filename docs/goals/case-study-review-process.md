# Lesly Case-Study Review Process

## Rule

Review and approve one public case study at a time. A case cannot move to `APPROVED` while its hero, gallery, related work, claims, or responsive presentation contains an unresolved duplicate or evidence problem.

## Per-Case Gate

1. **Identity and purpose** — confirm the project title, buyer value, visible asset family, and intended public role.
2. **Hero** — use one strong hero that is not repeated elsewhere on the same page.
3. **Gallery** — every image must add a distinct view or decision; no hero repeat, near-duplicate frame, ornamental filler, or unsupported technical board.
4. **Evidence** — distinguish visible direction from genuine topology, UV, rig, engine, slicer, source-file, or physical-output proof.
5. **Related work** — show three different visual families. Do not repeat the current case's hero, character family, or another related-card image.
6. **Responsive review** — verify desktop and mobile composition, natural image ratio, readable captions, no overflow, and a visible contact path.
7. **Approval** — record browser evidence and only then advance to the next case.
