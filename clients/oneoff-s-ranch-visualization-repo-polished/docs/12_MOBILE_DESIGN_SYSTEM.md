# Mobile Proposal Design System

This repo intentionally uses the more polished first-generation mobile presentation language:

- mobile-first shell: full mobile on phones, with permission to expand gracefully up to 1024px for wider review when confirmed by the user
- cinematic hero with gradient overlay
- fixed top brand bar and bottom pill navigation
- Georgia serif display headings + system sans body
- dark walnut / brass / ivory / paper palette
- reusable panels, paper cards, KPI grid, asset grid, mood cards, timeline, tabs, quotes, role card, scope cards, portal prototype, and lightbox
- high-contrast labels: dark labels on paper cards, brass labels on dark sections

## Content hierarchy

1. Hero: reframes the work as a client approval system.
2. Vision: establishes director-level strategic thesis.
3. Case: grounds the story in the S-RANCH bar-window scenario.
4. Technical: verifies the 2+2 sliding bifold mechanism and depth correction.
5. Assets: shows visuals only when each has an explicit job.
6. Portal: demonstrates private commission page and tap-to-comment workflow.
7. Workflow / Scope / Role / Questions: turns the concept into a clear next-step proposal without pricing discussion.

## Critical technical rule

Do not represent the lower window as four ordinary swing doors. It is two sliding bifold pairs:

- panels 1+2 fold and slide/park left
- panels 3+4 fold and slide/park right
- when left side only is open, show exactly one folded stack on the left and keep the right pair closed
- behind the opening, show a shallow-to-moderate recessed display niche, not a full kitchen or walk-in room
