# Mobile Presentation Design System Skill

Use this skill when editing the client-facing HTML presentation.

## Visual language

Use the polished phone-canvas design system:
- dark walnut / brass / ivory palette
- cinematic hero with image overlay
- fixed topbar + bottom pill navigation
- serif display headings, system sans body
- reusable cards: panel, paper, KPI, timeline, tabs, asset grid, mood cards, quote, role card, scope cards, video slot, portal prototype

## Rules

- Mobile-first. A narrow phone view is the primary experience, but when the user confirms wider delivery, let the presentation expand gracefully up to 1024px instead of trapping it in a phone-only frame.
- Maintain strong label contrast: paper cards use dark labels; dark sections use brass labels.
- Do not flatten all content into simple stacked text blocks.
- Every visual must have a job: emotional approval, mechanism clarity, depth validation, material direction, or client portal demonstration.
- Keep the S-RANCH technical constraints visible in every major revision.
- Keep send-ready pages free of "placeholder", "internal", "draft", "archive", and "questionnaire" language.
- Do not link to `site/alignment-questionnaire.html`, archived HTML pages, raw intake files, or internal audit docs from the client-facing page.
- Label unconfirmed finish/material examples as concept direction.
- Browser-check at mobile width before finalizing: hero readability, bottom nav usability, image loading, tap interactions, no console errors, and no broken images.
