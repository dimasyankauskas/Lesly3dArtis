# Skill: Proposal Writer

Use this skill to write client-facing messages and proposal pages.

## Tone
Respectful, strategic, warm, practical.

## Core argument
Move beyond a single 2D-to-3D render into a visual approval system that creates more value for One Off, designers, and high-end clients.

Do not insult One Off's drawings or existing workflow. Use wording like: "Technical drawings are essential for fabrication; this adds a client-facing approval layer."

- `docs/06_FINAL_PITCH_MESSAGE.md`

## Structure
1. Respect current drawings.
2. Show technical understanding.
3. Reframe as client experience.
4. Propose pilot.
5. Offer scalable workflow.

## Final ask

- Primary ask: discuss a focused Visual Strategy Pilot.
- Do not include pricing, dollar ranges, package ladders, retainers, or estimate calculators in the presentation.
- Keep larger package, portal, and retainer language as optional future workflow directions, not priced offers.
- Include a clear next step: confirm key details and discuss the right pilot scope this week.

## Client-safe boundaries

- Concept/client-approval visuals are not fabrication drawings.
- Final finishes, colors, lattice/infill, and fabrication details remain with One Off until confirmed.
- Do not mention internal questionnaires, archived pages, or raw intake files in the client send message.
