# 3D character artist portfolio — segment context

Use this skill when generating or editing any of: hero copy, about, services grid, package descriptions, case study breakdowns, process steps, contact form labels, social posts, or resume/CV for a 3D character artist's portfolio. The default Codex output for "character artist portfolio" tends to be generic ("passionate creator," "bringing imagination to life," logo work, NFT mention). This skill enforces the segment's actual conventions.

## Positioning — the load-bearing rule

A 3D character artist's portfolio lives or dies on **specialism**. The default Codex tendency is to make the artist sound like a generalist ("I do 3D modeling, illustration, concept art, animation, game design, NFT, logo work..."). That kills conversion. The correct frame:

- Lead with **one flagship offer**, not a buffet.
- Frame AI as a **workflow enhancement**, never the product.
- Sell **production-ready deliverables**, not "creative vision."
- "Character-first" thinking: the character is the asset, the artist is the producer.

### Test: if a recruiter reads the hero and cannot say in five seconds what this artist does and for whom, the copy is wrong.

## Specialties (priority order for FT + freelance mix)

When a project lists multiple specialties, lead with the first. The others go in descending order; later items should never be the dominant copy.

1. **Concept / AI Reference → 3D Character** (flagship — the highest-leverage offer for both game studios and indie clients)
2. **Game-Ready Stylized Characters** (retopo, UVs, textures, engine-ready exports)
3. **Brand Mascot for Startups / AI Products**
4. **Avatar / VTuber / Streamer Character**
5. **Roblox / UGC / Stylized Asset Design**
6. (Optional) **AI-Assisted Visual Development** — only as workflow, never as the headline

## What recruiters actually look for (FT hire rubric)

When the goal is a US game-studio FT character-artist role, the portfolio must show **all** of these, not just final renders:

- **Turnarounds** — front, side, back, three-quarter views
- **Wireframes + retopo** — proves the topology is real, not a subdivision-surface ZBrush dump
- **Texture/material sheets** — PBR, UV layout, material breakdown
- **Before/after** — sketch/AI reference → final 3D
- **Engine-ready exports** — FBX, OBJ, BLEND, with notes for Unity/Unreal
- **Pose render set** — 2-4 poses per strong character, expressive range
- **3 detailed case studies** — problem, direction, exploration, execution, final, deliverables
- **Process visibility** — sculpt → retopo → UV → texture → shade → render
- **Tool fluency** — ZBrush, Maya, Blender, Substance Painter, Marmoset Toolbag
- **Style range**, but **cohesive** — 1-2 styles max in any one piece, no tonal whiplash

Missing wireframes or missing engine-export notes = serious downgrade. A beautiful sculpt with no retopo reads as "junior hobbyist" to a senior recruiter.

## Case study anatomy (use this structure for any project card → detail page)

Every case study must contain, in this order:

1. **Problem** — who is the client, what do they need the character for, what was blocking them
2. **Direction** — references, moodboard, style choice, why
3. **Exploration** — 2-4 concept variations with the choice rationale
4. **3D creation** — sculpt → retopo → UVs → texture → shade, with screenshots
5. **Engine-ready** — wireframe, polycount, export notes, what pipeline it slots into
6. **Final renders** — 3-5 presentation images, the "money shot" first
7. **Deliverables** — exactly what the client received (file list, formats, sizes)

A case study without the engine-ready section is incomplete. A case study without problem/direction is just a render dump.

## Package framing (for paid inquiries)

Use clear tiers, named with the buyer's outcome, not the artist's process:

- **Concept Sprint** — quick character concept exploration (1-2 directions, moodboard)
- **3D Character Starter** — single character, sculpt + retopo + basic texture
- **Game-Ready Character** — full retopo, UVs, PBR textures, engine-ready exports
- **Mascot System** — character + expression set + pose set + brand colorways
- **Avatar / VTuber Kit** — character + expression set + outfit variations
- **Pipeline Consulting** — character style guide, art-direction review, technical planning

Do not call a package "Basic / Standard / Premium" — name it after what the client gets.

## Red-flag patterns to avoid

These phrases and patterns will hurt the brand. Check every draft against this list:

- "Passionate 3D artist" / "creative visionary" / "bringing imagination to life" — every generic portfolio says this
- "I can do anything" / "all styles" / "no project too big or small" — signals generalist
- Logo design, NFT language, "metaverse character" without context
- Tool-list-as-positioning ("I use ZBrush, Maya, Blender, Substance, Marmoset, Houdini, Unreal, Unity...") — show the tools in case studies, do not lead with them
- "Years of experience" without proof of output
- Stock adjectives: stunning, breathtaking, vibrant, captivating, masterpiece (let the renders carry the weight)
- "Let's collaborate" / "Let's make something amazing together" — passive CTAs
- "Trusted by clients worldwide" without names
- Promotional triple ("vision, quality, and passion" / "speed, precision, and creativity")
- Em-dash overuse, rule-of-three, AI vocabulary words (delve, leverage, tapestry, testament, pivotal) — see humanizer skill

## Voice and tone

- **Confident, not boastful.** "I deliver" not "I strive to deliver."
- **Specialist, not exclusive.** "Character work is my focus" not "I only do characters."
- **Production-aware, not jargony.** "Game-ready topology" yes; "low-poly asset with optimized edge flow" maybe, only if the audience is technical.
- **Direct, not promotional.** State the deliverable, the timeline, the price range. No "inquiry for details" without an actual starting price.
- **First person, present tense.** "I sculpt, model, and texture" not "Lesly is a sculptor who..."

## CTAs that actually convert (character-artist sites)

Use one primary + one secondary on the hero. The primary must be specific:

- Primary: **"Start a project"** or **"Hire me for a character"**
- Secondary: **"View work"** or **"See process"**

Never use "Get in touch" / "Contact" / "Learn more" as the primary. Never use "Let's chat."
