## Rules

- Real Lesly work may be presented as portfolio proof only after the user or an authoritative repo note approves it.
- User direction in this goal clarified that Codex should create the V2 visuals. Original generated site concept assets may be used only when clearly labeled as generated/site-created visuals and not client work, shipped work, source 3D proof, topology, UV, texture, or handoff evidence.
- Do not use `../reserch/visuals/image*.png`; those are design/mockup references.
- Do not add fake project names, client names, stats, wireframes, UVs, texture sheets, engine targets, or deliverable filenames.
- If a fact is unknown, leave it blank and list the exact blocker in `LAUNCH_CHECKLIST.md`.
- Prefer `.webp` for final web assets.

## Ready-To-Replace Checklist

Only move a slot to real portfolio proof when all are true:

- The file is real Lesly work, an approved portrait, or a generated concept visual labeled as such.
- The file has permission for public portfolio use.
- The filename and target path are confirmed.
- The website copy around it does not imply unsupported facts.
- Any technical stats, file formats, tools, client/project names, and production claims are verified.
- Any environment, lighting, camera, HDRI, prop, viewer, or engine claim is verified and credited where needed.
